**INFRA-03: Add VRAM guard between sequential model evaluations.** Per assumption A1 in CONTEXT.md, each remote model evaluation is a separate SSH command launching a new Python process -- VRAM is freed by the OS when the process exits. The VRAM guard detects failed processes that left VRAM dirty.

1. Add a helper function `build_vram_probe_command(ssh_target: str) -> list[str]` that builds an SSH command to run `nvidia-smi --query-gpu=memory.used,memory.total,memory.free --format=csv,noheader,nounits` on the remote host. Return the result via `build_ssh_command()`.

2. Add a helper function `parse_vram_probe(runtime: dict) -> dict[str, Any]` that:
   - Parses the nvidia-smi CSV output (memory.used, memory.total, memory.free in MiB)
   - Returns `{"available": True, "used_mib": int, "total_mib": int, "free_mib": int, "utilization_pct": float}`
   - On parse failure, returns `{"available": False, "error": "..."}`

3. Add a constant `VRAM_SAFETY_THRESHOLD_MIB = 512` -- if more than this much VRAM is in use before loading a model, something is wrong (leaked from prior model or other process).

4. In the model evaluation loop (around line 1077, `for model_name in backend["models"]:`), insert a VRAM probe **before** each model evaluation (after `if backend_failure_stage is None:` check at line 1128):
   - Run `build_vram_probe_command` with `timeout=DEFAULT_TIMEOUT_PROBE`
   - Parse with `parse_vram_probe`
   - Record in `backend_shared_runtime[f"model_{model_slug}_vram_pre"]`
   - If `used_mib > VRAM_SAFETY_THRESHOLD_MIB`: set `failure_stage = "vram_dirty"` and skip to next model (do NOT abort the entire backend -- later models might succeed if the problem was transient)
   - Log the VRAM state in the model's runtime.json

5. **Test coverage additions** in `test_remote_embedding_backends.py`:
   - Test `parse_vram_probe` with valid CSV output (e.g., "256, 11264, 11008") returns correct parsed values
   - Test `parse_vram_probe` with empty/garbage output returns `{"available": False, ...}`
   - Test that `build_vram_probe_command` produces a well-formed SSH command
   - Test `run_command` timeout behavior: mock `subprocess.run` to raise `subprocess.TimeoutExpired`, verify the returned dict has `"status": "timeout"` and appropriate error message
   - Test `run_command` with timeout=None still works (backward compatibility)

Keep the VRAM guard as a safety check (log warning + skip model), not a hard pipeline abort. Per CONTEXT.md reversibility note: "can start with a warning and escalate to a gate."
